------------------------------------------------------------
EXPLANATION OF EACH FOLDER & FILE
------------------------------------------------------------

main.py
- Entry point of the entire application.
- Starts voice module, gesture module, UI, and central controller.

------------------------------------------------------------
config/
------------------------------------------------------------
command_map.json
- Maps gesture IDs and voice phrases to actual OS commands.

gesture_config.json
- Settings for gesture recognition (thresholds, sensitivity).

voice_config.json
- Parameters for voice engine, wake word, noise filtering.

app_settings.json
- General app preferences (theme, language, UI behavior).

Purpose: Centralized settings that can be edited without touching code.

------------------------------------------------------------
command_controller/
------------------------------------------------------------
controller.py
- Receives events from gesture and voice modules.
- Decides which command to execute.

executor.py
- Actually performs system actions (keyboard, mouse, window control).

logger.py
- Logs all executed commands, errors, and events.

__init__.py
- Allows importing this folder as a package.

Purpose: The "brain" of the entire app.

------------------------------------------------------------
gesture_module/
------------------------------------------------------------
gesture_detector.py
- Detects swipe directions, finger counts, and gesture IDs.

hand_tracking.py
- Raw MediaPipe or OpenCV hand tracking.

gesture_utils.py
- Math helpers, smoothing, landmark normalization.

models/
- Future ML models for advanced gesture recognition.

__init__.py
- Makes this folder a Python package.

Purpose: Everything related to hand gestures stays here.

------------------------------------------------------------
voice_module/
------------------------------------------------------------
voice_listener.py
- Constantly listens for audio input.

stt_engine.py
- Converts speech → text using Vosk/Whisper/etc.

voice_utils.py
- Cleans text, filters noise, handles keywords.

keywords/
- Custom word lists or trigger phrases.

__init__.py
- Makes this folder importable.

Purpose: Keeps voice functionality isolated and clean.

------------------------------------------------------------
ui/
------------------------------------------------------------
main_window.py
- The primary window of the application.

gesture_editor.py
- UI for adding/editing gestures and associated commands.

voice_editor.py
- UI for adding/editing voice commands.

assets/
- Icons, GIF previews, stylesheets for UI.

__init__.py
- Lets you import the UI package.

Purpose: Separate UI from logic so designers and developers can work independently.

------------------------------------------------------------
data/
------------------------------------------------------------
gesture_recordings/
- Stores user-created gesture recordings (JSON or GIF).

user_profiles/
- User-specific settings.

logs/
- Log files generated by logger.py.

Purpose: All data generated during use is saved here.

------------------------------------------------------------
utils/
------------------------------------------------------------
threading_utils.py
- Helpers for running gesture + voice listeners concurrently.

event_bus.py
- Central message queue between modules.

file_utils.py
- Safe loading/saving of JSON and data.

system_utils.py
- OS checks, environment helpers.

Purpose: Shared helpers used across multiple modules.

------------------------------------------------------------
SUMMARY
------------------------------------------------------------
This structure ensures:
- Clean separation of voice, gesture, UI, and command logic.
- Easy collaboration among team members.
- Scalable architecture for future expansion.
- Professional, industry-style project organization.
